来自英语专业的顺利毕业 刚刚入门

大家都知道，学编程是需要一点英语能力的，或许你会问，英语零基础也可以学嘛，那么作为英语专业的一名准大三学生，我可以很明确地告诉你，完全没问题！

还记得刚加入训练营，做第一天作业的时候，自恃是英语专业的，看到全英文的官方文档，第一反应就是想着靠自己的英文水平去看懂，去学习，结果当然是一败涂地，先不说能不能坚持看完,光是自己理解的错误就不计其数，原本几个小时就能搞定的东西，花了二十多个小时才完成，效率是何等的低呀。还好第二天发现了自己的问题，及时改正，用了谷歌浏览器，效率一下子提高了，所以你看吧，其实看全英文的官方文档，用google浏览器的翻译功能就比我这个英语专业的效率高多了，那么有那么一点英语能力是不是有用的，是有的，有时候可能机器翻译的不够地道，用中英文来对照地话更轻松一些，还有就是在编程的时候能更好地理解是什么意思吧。说白了，有一定的英语水平效率可能会高上一些，零基础的话，其实也不会遇到多大的困难，相反，并不是用英语去学编程，很多时候反而是用编程顺带学了英语了。

那么进入正题了，这“14天”的自学历程到底给我带来了什么。

1.“过早引用”
很幸运知道了这个概念：所有理解起来困难、学习起来困难的知识体系，都具备大量的“过早引用“，所以必须学习处理这些”过早引用“，才是你习得自学能力的最基本条件。作为一个从零开始学一个知识体系的学生来说，最痛苦的事情莫过于“过早引用“的概念了。每次遇到大量的”过早引用“，扑面而来的挫折感太容易打击自信了，这14天的学习没有一天不是这种情况，然而坚持下来之后，你就会发现，即使痛苦，可那只是”过早引用“的概念，而并不是你学不会的概念，只是花的时间不够，重复地不够。经历过这个，并且坚信这个之后，自学任何东西，即使再痛苦，也不会那么容易放弃了。

2.了解知识框架，清楚学习目的
还记得在自学训练营报名的界面有一个很清楚的课程大纲，你会知道你学习的是什么，你将会学习到什么以及还有什么没学的，我自认为这是很重要的，学习前，在脑中有一个明确的知识框架，学到的东西可以有序地放进去，就像一个架子一样，学到的东西放进属于它的位置，还没有学到的东西也有它自己空着的位置。
学习中，就像每次去完成训练营的作业之前，都会有一个学习目的在，那么我们在学习的过程中，可以把学习目的更加地细化，把问题拆分，拆分成一个个的小问题，解决这些问题，再组装起来，解决大的问题就简单多了，效率也明显上升。
3.学习记录加总结
自学营还有一个特别有意思的地方就是在学习完之后，在糖果app内发布笔记打卡，对自己的学习内容和学习感受进行总结。温故而知新，其实大家都知道，只是养成这个习惯没那么容易。开启下一次作业的标志是进行打卡总结，这样的方式很有效地帮助习惯的养成。

4.注意力更集中了
一个人一天的注意力是有限的，注意力分散，学习效率就会低很多。2个小时注意力分散式的学习可能还比不上1个小时集中注意力学习的效率。经过这14天的学习，学会了感受自己注意力的状况，累了，注意力下降了就休息，一直保持高强度注意力集中学习的状态。

5.耐心
即使学习效率再高，也要一定的时间才能看到成果，中途放弃，不如不学，14天的时间，也仅仅是入门，要时刻告诉自己，保持耐心，不要急躁，让时间告诉你答案。
